🚨 BILL_TO ADDRESS STRUCTURE RULES - IMPORTANT 🚨
 
The bill_to structure MUST remain exactly in this format:
 
"bill_to": {
  "address": {
    "city": "string",
    "line1": "string",
    "line2": "string",
    "state": "string",
    "country": "string",
    "postal_code": "string"
  }
}
 
DO NOT:
- add extra keys
- change key names
- change hierarchy
- merge fields together
- return raw multiline address
 
Always preserve the exact bill_to.address structure.
 
Field Extraction Rules:
 
city:
- Extract only city name
- Never include postal code with city
- Remove commas and OCR artifacts
 
line1:
- Extract primary street/building/location details
- Include road, street, industrial estate, locality, village if present
- Do NOT include city/state/country/postal_code
 
line2:
- Extract secondary address details if available
- Examples:
  - Taluk
  - District
  - Area
  - Additional location details
- If unavailable return empty string ""
 
state:
- Extract normalized state value or state code only
- Never combine city/state/postal_code together
 
country:
- Extract normalized country value/code only
 
postal_code:
- Extract only valid postal/PIN code
- Return only postal code value
- Never include city/state/country in postal_code
 
🚨 POSTAL CODE EXTRACTION RULES 🚨
 
Postal codes may:
- appear before city
- appear after city
- appear inside multiline addresses
- appear with commas
- appear with OCR spacing issues
- appear merged with locality names
 
Always identify postal_code contextually from the FULL address block.
 
Examples:
- 641021 COIMBATORE
- Bangalore - 560001
- 641201 KALLAPALAYAM, SULUR TALUK
 
Valid postal code handling:
- Indian postal codes are typically 6 digits
- Preserve exact postal code value
- Never confuse PO numbers, phone numbers, GST numbers, invoice numbers, or quantities as postal codes
