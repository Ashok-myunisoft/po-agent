- Dynamically detect postal-related patterns including:

  "postal_code"

  city, state, postal_code, and country

- Do not duplicate postal code values across multiple fields unless explicitly present in the original document

- Avoid storing full address lines inside:
  "postal_code"

- Never return:
  "postal_code": "NULL"
  if any valid postal identifier exists anywhere in the address block

- Handle international and region-specific address formats dynamically without relying on fixed templates
